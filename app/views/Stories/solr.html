Information Architecture with Solr and Lucene

Working as a Principal Research Scientist at Battelle Memorial Institute, Developer Dan Funk revamped a search engine for a large collection of research documents which had been exhibiting speed and scalability issues. Using the Apache Lucene™ library, Dan developed a powerful, full-text search engine, and OCR (optical character recognition) workflow process to create a polished, accessable, and intuitive interface to a large body of government documents. Ultimately all search times were reduced to a few milliseconds, some from a previous high of as much as 30 seconds. While the previous system had to be run on large-scale servers, the new search system could comfortably run on a portable device.

Solr is an open source search platform—imagine your own custom search engine—just as fast and intuitive as Google, but configured to do exactly what you want, and to search only the content that interests you. Solr provides hit highlighting and faceted search, on top of the phenomenally fast Lucene full-text search library. Solr scales. Dynamic clusters, distributed search, and index replication mean you will not outgrow its capabilities.
